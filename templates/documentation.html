Where to Start

The NQSync QKD simulator is a web-based simulation platform designed for building and testing quantum key distribution (QKD) networks. To begin, click the “Launch Simulator” button which opens a canvas interface.

Working with the Canvas

  • Add Nodes: Create devices (systems/ports) on the canvas that can be connected with links and configured with QKD protocols.
  • Simulate Network: Runs the simulation based on the current topology and configured links. Opens the results page showing metrics like QBER.
  • Reset: Clear the canvas to start fresh.

Double-click a node to open the configuration panel and create a link.

  • Node Name: Name of the selected system.
  • From: Auto-filled with the selected node.
  • To: Choose the destination node to connect.
  • Protocol: Choose between DPS QKD or CoW QKD.
  • Cancel: Closes the panel without saving.
  • Delete Node: Removes the selected node (only if not linked).
  • Configure: Opens the parameter form and adds a link once configured.

When you click Configure, you will be directed to one of the following forms depending on the selected protocol—either DPS QKD or COW QKD. These forms are pre-filled with default values, which you can modify as needed.

Once you click Save Configuration, you’ll return to the canvas where a colored link will appear between your selected systems: blue for DPS and yellow for CoW.

You can then click Simulate Network to view performance metrics and results. To delete a link, left + right click simultaneously on it and confirm when prompted.

Available Protocol Parameters

Common Parameters (DPS & CoW)

  • distance_km – Distance between two nodes (in kilometers) over which quantum key distribution is performed.
  • fiber_type – Type of optical fiber used in the channel. Can be "underground" or "aerial", affecting signal loss.
  • mu – Mean photon number per pulse. Represents the average number of photons sent by Alice per pulse. Default is 0.2 for DPS and 0.1 for CoW.
  • pulse_rate – Pulse repetition rate (in Hz) at which photons are sent by Alice.
  • det_eff – Detector efficiency. Probability (between 0 and 1) that a photon is successfully detected.
  • dark_rate – Dark count rate of the detector (counts per second). Represents false clicks due to noise or thermal effects.
  • dead_time – Dead time of the detector (in seconds). It is the interval after each detection during which the detector is inactive and cannot record further events.
  • visibility – Interference visibility, indicating the quality of phase coherence. Values closer to 1 imply ideal interference and lower error rates.
  • num_pulses – Number of photon pulses sent by Alice during the simulation. Affects how statistically stable the results are.

DPS QKD-Specific Parameters

  • disclosure_rate – The fraction of the raw key disclosed for QBER estimation. Higher values improve accuracy but reduce usable key length.
  • compression_ratio – The factor with which the key is shortened during privacy amplification to remove leaked information.

CoW QKD-Specific Parameters

  • error_correction_efficiency – Efficiency of the error correction algorithm used in post-processing. Typically between 0.9 and 1. Higher values preserve more of the key.

Supported Protocols

  1. CoW QKD Protocol
  2. DPS QKD Protocol

CoW QKD Protocol

This protocol uses one-way coherent pulses and simple detection logic. Key calculations include:

  • Alpha: 0.20 (underground), 0.30 (aerial)
  • Loss: alpha × distance
  • Transmittance: 10-loss/10
  • Detection probability is derived from signal and dark count rates.
  • QBER is calculated based on bit errors from non-decoy states.

DPS QKD Protocol

This protocol uses phase differences between consecutive pulses:

  • Keys are extracted based on detected phase differences.
  • QBER is the ratio of bit mismatches at comparable positions.
  • Final key rate = sifted rate × (1 - disclosure_rate) × (1 - compression_ratio).

Understanding Simulation Result Metrics

  • Distance_km — Distance between sender(Alice) and receiver(Bob) in kilometers over the optical fiber.
  • Fiber_Type — "underground" or "aerial" fiber which affects attenuation (0.20 for underground and 0.30 dB/km for aerial).
  • Channel_Loss_dB — loss_dB = α × distance — Total loss of signal power over the fiber due to attenuation.
  • Transmittance — T = 10-loss_dB / 10 — Fraction of signal power that successfully passes through the fiber.
  • Signal_Probability_per_Pulse — Psignal = μ × T × η — Probability that a signal photon sent by Alice is successfully detected by Bob.
    Where:
      μ = mean photon number per pulse
      T = transmittance
      η = detector efficiency
  • Detection_Probability — Pdetect = Psignal + (dark_rate / pulse_rate) — Total probability that a pulse results in detection (including both signal and noise).
  • QBER — QBER = bit_errors / comparable_bits — Quantum Bit Error Rate, the fraction of incorrect bits shared between Alice and Bob.
  • Bit_Errors — Number of mismatched bits between Alice and Bob during key comparison.
  • Key_Match_Ratio — 1 - QBER — Percent of matching bits in valid positions.
  • Raw_Key_Rate_bps — Rraw = (pulse_rate × Pdetect) / (1 + pulse_rate × Pdetect × dead_time) — Initial bit rate of key generation based on detection probability.
  • Sifted_Key_Rate_bps — Rsifted = 0.5 × Rraw — Rate of key bits retained after sifting.(~50% of raw key is usable.)
  • Final_Key_Rate_bps — Rfinal = Rsifted × (1 - disclosure_rate) × (1 - compression_ratio) — Usable key rate after error estimation and privacy amplification in DPS QKD.
    Note for CoW QKD: The final key rate is calculated using a different formula: Rfinal = Rsifted × (1 - error_correction_efficiency × QBER)
    This accounts for error correction efficiency instead of disclosure and compression rates.
  • Visibility — Depicts interference quality. Affects how accurately Bob can infer Alice’s bit from phase differences.
  • Raw_Key_Length_Alice — Total number of bits Alice attempted to encode and transmit.
  • Raw_Key_Length_Bob — Total number of bits detected and stored by Bob.
  • Num_Detections — Total detections, including both signal-based and dark counts.
  • Signal_based_Detections — Count of detections caused by actual signal photons.
  • Dark_only_Detections — Detections that occurred due to detector noise or false positives (not actual signal).
  • Comparable_Bits — Bits that are eligible for comparison between Alice and Bob to compute QBER.
